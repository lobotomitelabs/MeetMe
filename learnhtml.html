Part One: HTML
the bones of the web — structure & content
HTML · Chapter 01

What Even Is HTML?

✦

HTML stands for HyperText Markup Language. It's not really a "programming language". It's more like a set of labels you put around your content to tell the browser what everything is.

Think of it like annotating a manuscript. You're not making the words do math. You're saying "this is a heading," "this is a paragraph," "this is a link." The browser reads those labels and renders everything into a page.

🌹 no pressure vibes: HTML is one of the most forgiving things to learn. Typos usually just make things look a little weird — they don't explode. You can't break the internet with a bad <p> tag.

Every single webpage you've ever visited! Every itch.io game page, every Neocities shrine, every fanfic archive — starts with HTML. It's the bones of the web.

HTML · Chapter 02

Anatomy of a Tag

✦

HTML is built from tags. A tag is a word wrapped in angle brackets: <like this>. Most tags come in pairs:

basic element
<p>This is a paragraph of text.</p>
<!--  ↑ opening tag    ↑ closing tag (has a /) -->

Tags can also have attributes extra info inside the opening tag:

with attributes
<a href="https://itch.io" target="_blank">visit itch.io!</a>
<!--   ↑ attr name   ↑ attr value (always in quotes) -->
💀 void elements (self-closing): some tags don't wrap content — they just are a thing, like <img> or <br>. No closing tag needed.
HTML · Chapter 03

The Shape of a Page

✦

Every HTML document follows the same skeleton:

index.html
<!DOCTYPE html>        <!-- "this is html5" -->
<html lang="en">

  <head>
    <!-- invisible to users: metadata, title, CSS links -->
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>My Cool Page</title>
    <link rel="stylesheet" href="style.css">
  </head>

  <body>
    <!-- everything users actually see goes here -->
    <h1>hello world</h1>
    <script src="script.js"></script>  <!-- JS goes at end of body -->
  </body>

</html>
🌙 the viewport meta tag makes your page not look tiny and zoomed out on phones. always include it.
HTML · Chapter 04

Text & Headings

✦

HTML has six heading levels (<h1> through <h6>). <h1> is the most important — use it once per page, as the main title. The others are section/subsection headers.

text elements
<h1>Page Title (use once)</h1>
<h2>Section Header</h2>
<h3>Subsection</h3>

<p>A normal paragraph. Can be long. Wraps naturally.</p>

<p>Make text <strong>bold</strong>, <em>italic</em>,
or <strong><em>both</em></strong>.</p>

<p>You can also do <mark>highlighted</mark>, 
<del>strikethrough</del>, <small>small text</small>,
and <code>inline code</code>.</p>

<blockquote>A quoted passage, indented by the browser.</blockquote>

<br>  <!-- line break — void element -->
<hr>  <!-- horizontal rule (divider line) -->
✨ don't skip heading levels just for style — screenreaders use them to navigate the page. Change how they look with CSS instead.
HTML · Chapter 05

Links & Images

✦

Links use <a> (anchor). href is where it goes. target="_blank" opens in a new tab.

links
<a href="https://itch.io" target="_blank">new tab link</a>
<a href="about.html">relative link (same site)</a>
<a href="#section-id">jump to section on this page</a>
<a href="mailto:hi@example.com">email link</a>

Images use <img> — void element, no closing tag. alt text is required for accessibility.

images
<img src="cat.png" alt="a small gray cat">

<!-- responsive (CSS will control actual size) -->
<img src="banner.jpg" alt="site banner" width="800" height="300" loading="lazy">

<!-- clickable image (link wrapping an img) -->
<a href="https://itch.io">
  <img src="logo.png" alt="go to itch.io">
</a>
HTML · Chapter 06

Lists

✦

Three types: unordered (bullets), ordered (numbers), and definition lists (term + description pairs). Each item is always a <li>.

lists
<ul>  <!-- unordered -->
  <li>mushrooms</li>
  <li>cryptids</li>
</ul>

<ol>  <!-- ordered (numbered) -->
  <li>open godot</li>
  <li>make something cool</li>
</ol>

<dl>  <!-- definition list -->
  <dt>HTML</dt>
  <dd>the structure of a webpage</dd>
  <dt>CSS</dt>
  <dd>the style of a webpage</dd>
</dl>

<!-- lists can be nested! -->
<ul>
  <li>games
    <ul>
      <li>godot games</li>
      <li>browser games</li>
    </ul>
  </li>
</ul>
HTML · Chapter 07

Divs & Spans

✦

<div> is a block container takes full width, starts on a new line. For sections, cards, wrappers.

<span> is an inline container sits inside text. For styling a single word or icon mid-sentence.

divs & spans
<div class="card">
  <h2>My Card</h2>
  <p>some content in a styled box</p>
</div>

<p>this word is <span class="glow">glowing</span> in pink.</p>
🌹 class vs id: class can repeat on multiple elements. id must be unique per page. Use class for styling, id for JS targeting or page anchors.
HTML · Chapter 08

Semantic HTML

✦

Semantic tags are like <div> they're containers — but they have meaning. They tell browsers, search engines, and screenreaders what role each part of your page plays.

semantic structure
<header>
  <!-- site logo, main nav, page title -->
  <nav>
    <a href="/">home</a>
    <a href="about.html">about</a>
  </nav>
</header>

<main>
  <!-- primary unique content of the page -->
  <article>
    <!-- self-contained content: a blog post, a game listing -->
    <h1>My Game</h1>
    <p>a cozy cryptid farming rpg</p>
  </article>

  <section>
    <!-- thematic group within the page -->
    <h2>Features</h2>
  </section>

  <aside>
    <!-- sidebar, related links, extra info -->
  </aside>
</main>

<footer>
  <!-- copyright, social links, small print -->
</footer>
🌙 why bother? a screenreader user can jump straight to <main> and skip the nav. Google understands your <article> is important content. Semantic HTML = better accessibility + better SEO, for free.
HTML · Chapter 09

Forms & Inputs

✦

Forms let users send you data — login screens, contact forms, search bars, game settings. The <form> wraps everything, and <input> is the actual field.

form elements
<form>

  <!-- text input -->
  <label for="name">your name</label>
  <input type="text" id="name" placeholder="mysterious traveler...">

  <!-- email input (validates format) -->
  <input type="email" placeholder="you@example.com">

  <!-- multi-line text -->
  <textarea rows="4" placeholder="your message..."></textarea>

  <!-- dropdown -->
  <select>
    <option value="goblin">goblin</option>
    <option value="cryptid">cryptid</option>
  </select>

  <!-- checkbox -->
  <input type="checkbox" id="subscribe">
  <label for="subscribe">send me cursed updates</label>

  <!-- range slider -->
  <input type="range" min="0" max="100">

  <button type="submit">send it ✦</button>

</form>
✨ always pair <label> with inputs using for="id". it makes them clickable AND accessible — screenreaders read the label out loud when focusing the input.

HTML · Chapter 11

Attributes Deep Dive

✦

Some attributes work on any element. Others are specific to certain tags. Here's a rundown of the most useful ones:

global attributes
<!-- class: for CSS styling (can repeat) -->
<div class="card highlight">two classes at once</div>

<!-- id: unique identifier (one per page) -->
<section id="about">...</section>

<!-- style: inline CSS (avoid for big projects) -->
<p style="color: pink; font-size: 20px;">pink text</p>

<!-- title: tooltip on hover -->
<abbr title="HyperText Markup Language">HTML</abbr>

<!-- hidden: hides element (JS can show it later) -->
<div hidden>you can't see me yet</div>

<!-- data-*: custom data for JavaScript -->
<button data-item-id="potion" data-qty="3">buy</button>

<!-- tabindex: control keyboard focus order -->
<div tabindex="0">now keyboard-focusable!</div>

CSS · Chapter 01

What Is CSS?

✦

CSS stands for Cascading Style Sheets. It's what makes things look like things. Colors, fonts, sizes, layouts, animations — all CSS.

You write CSS rules that say: "find this element, apply these styles." You can write CSS in a separate .css file (best for real projects), inside a <style> tag in your HTML, or inline with the style attribute.

css rule anatomy
p {                         /* selector: targets all <p> tags */
  color: #e8a0b0;           /* property: value; */
  font-size: 18px;
  line-height: 1.6;
}
🔮 the "cascading" part means styles flow down and can override each other. More specific rules win. Later rules override earlier ones. We'll cover this in the Cascade section.
CSS · Chapter 02

Selectors

✦

Selectors are how you target elements. They range from simple to powerful:

selector types
/* element selector — targets all of that tag */
h2 { color: pink; }

/* class selector — targets class="card" */
.card { background: #1a0d12; }

/* id selector — targets id="hero" */
#hero { font-size: 3rem; }

/* combine: p tags inside .card */
.card p { color: #aaa; }

/* direct child only */
.nav > a { padding: 8px; }

/* multiple selectors at once */
h1, h2, h3 { font-family: Georgia, serif; }

/* pseudo-class: only on hover */
button:hover { background: var(--rose); }

/* pseudo-class: first item in a list */
li:first-child { font-weight: bold; }

/* attribute selector: inputs of type text */
input[type="text"] { border: 1px solid pink; }
CSS · Chapter 03

The Box Model

✦

Every HTML element is a box. The box model describes the layers around it, from inside out: content → padding → border → margin.

margin
border
padding
content
box model properties
.card {
  /* padding: space INSIDE the box */
  padding: 20px;              /* all sides */
  padding: 10px 20px;         /* top/bottom  left/right */
  padding: 8px 16px 12px 16px; /* top right bottom left */

  /* margin: space OUTSIDE the box */
  margin: 16px 0;             /* same shorthand as padding */
  margin: 0 auto;             /* horizontally centered! */

  /* border */
  border: 1px solid #5c2d3e;
  border-radius: 8px;          /* rounded corners */

  /* width & height */
  width: 300px;
  max-width: 100%;             /* responsive: never wider than parent */

  /* box-sizing (ALWAYS set this globally!) */
  box-sizing: border-box;     /* padding is included IN the width */
}
🔮 box-sizing: border-box is one of the most important CSS tricks. Without it, adding padding makes your box wider than expected. Most sites set it globally with * { box-sizing: border-box; }.
CSS · Chapter 04

Colors & Backgrounds

✦
color formats
.element {
  /* hex — most common */
  color: #c4607a;
  color: #c47;          /* shorthand for #cc4477 */

  /* rgb / rgba (a = alpha/opacity) */
  color: rgb(196, 96, 122);
  color: rgba(196, 96, 122, 0.5);  /* 50% transparent */

  /* hsl (hue, saturation, lightness) */
  color: hsl(345, 45%, 57%);

  /* named colors */
  color: hotpink;
  color: transparent;

  /* backgrounds */
  background-color: #1a0d12;
  background-image: url("texture.png");
  background-size: cover;         /* fill the box */
  background-position: center;

  /* gradients */
  background: linear-gradient(to bottom, #1a0d12, #0f080b);
  background: radial-gradient(circle, #c4607a, #1a0d12);
}

Typography

text styling

@import url('https://fonts.googleapis.com/css2?family=IM+Fell+English&display=swap');

body {
  font-family: Georgia, 'Times New Roman', serif; /* fallback stack */

  color: #e8a0b0;
}

  font-family: 'IM Fell English', serif;
  font-size: 3rem;       /* rem = relative to root font-size */
  font-weight: bold;
  letter-spacing: 0.05em;
  text-shadow: 0 0 20px rgba(196,96,122,0.6);
}

p {
  font-size: clamp(14px, 2vw, 18px); /* responsive font size! */
  text-align: left;   /* left / center / right / justify */
  text-decoration: none;
  text-transform: uppercase; /* none / uppercase / lowercase / capitalize */
}
CSS · Chapter 06

Flexbox

✦

Flexbox is how you lay things out in a row or column. It's one of the most powerful CSS tools and you'll use it constantly.

flexbox
.container {
  display: flex;

  /* direction */
  flex-direction: row;          /* row (default) | column */

  /* alignment on the MAIN axis (horizontal if row) */
  justify-content: center;      /* flex-start | center | flex-end | space-between | space-around */

  /* alignment on the CROSS axis (vertical if row) */
  align-items: center;          /* flex-start | center | flex-end | stretch */

  /* wrapping (when items don't fit) */
  flex-wrap: wrap;

  /* gap between items */
  gap: 16px;
}

.item {
  flex: 1;                      /* grow to fill space equally */
  flex-shrink: 0;               /* don't shrink below natural size */
  align-self: flex-end;        /* override align for just this item */
}
🔮 centering something perfectly: display: flex; justify-content: center; align-items: center; on the parent. that's it. that's the whole trick.
CSS · Chapter 07

The Cascade & Specificity

✦

When multiple CSS rules target the same element, specificity decides who wins. Higher specificity = higher priority.

specificity ranking
/* specificity from LOWEST to HIGHEST: */

p         { color: blue; }     /* element: 1 point */
.text     { color: green; }    /* class: 10 points */
#intro    { color: red; }      /* id: 100 points */
/* inline style="color:pink"        1000 points */
/* !important                       OVERRIDE ALL (avoid!) */

/* combining adds up: */
.card p   { color: pink; }     /* 10+1 = 11 points */
.card.featured p { color: gold; } /* 10+10+1 = 21 points — wins! */

/* when specificity is EQUAL, later rule wins: */
p { color: blue; }
p { color: pink; }  /* pink wins — it's later in the file */
CSS · Chapter 08

CSS Custom Properties (Variables)

✦

CSS variables let you define values once and reuse them everywhere. Change the variable and everything updates. Perfect for theming!

css variables
/* define on :root to make them global */
:root {
  --rose: #c4607a;
  --void: #0f080b;
  --font-body: Georgia, serif;
  --spacing-md: 16px;
}

/* use with var() */
body {
  background: var(--void);
  font-family: var(--font-body);
}

button {
  background: var(--rose);
  padding: var(--spacing-md);
}

/* you can even set a fallback value */
.thing {
  color: var(--text-color, #ffffff);  /* falls back to white */
}
CSS · Chapter 09

Transitions & Animations

✦

Transitions smoothly animate between two states (e.g. hover). Animations loop and run on their own with keyframes.

transitions & animations
/* TRANSITION: smooth state changes */
button {
  background: #c4607a;
  transition: background 0.2s ease, transform 0.2s;
}
button:hover {
  background: #e8a0b0;   /* smoothly fades to this */
  transform: scale(1.05);
}

/* ANIMATION: runs automatically */
.star {
  animation: twinkle 3s ease-in-out infinite;
}

@keyframes twinkle {
  0%, 100% { opacity: 0; }
  50%       { opacity: 1; }
}

/* floating effect */
@keyframes float {
  0%, 100% { transform: translateY(0); }
  50%       { transform: translateY(-10px); }
}

/* common transform functions: */
/* translateX/Y(px), scale(n), rotate(deg), skew(deg) */

JS · Chapter 01

What Is JavaScript?

✦

JavaScript is the programming language of the web. HTML is structure, CSS is style, and JS is behavior — making things actually do stuff when you interact with them.

Clicking a button and having something happen, menus opening and closing, loading new content, running a game — all JavaScript. It's also how you build Discord bots (Node.js), game logic in Godot-adjacent tools, and interactive apps.

⚡ you link a JS file with <script src="script.js"></script> at the end of your <body>. Or write it inline in a <script> tag. You can also open your browser DevTools console (F12) and run JS live right there.
JS · Chapter 02

Variables & Data Types

✦
variables
// let: can be reassigned
let playerName = "mysterious traveler";
playerName = "clio";  // fine!

// const: can't be reassigned (use this by default)
const maxHealth = 100;

// DATA TYPES:
const name     = "clio";          // string
const health   = 85;             // number
const alive    = true;           // boolean (true/false)
const nothing  = null;           // intentionally empty
const items    = ["sword", "potion", "map"];  // array
const player   = {                // object
  name: "clio",
  health: 85,
  level: 12
};

// access object properties:
console.log(player.name);      // "clio"
console.log(player["level"]);  // 12

// access array items (0-indexed):
console.log(items[0]);           // "sword"
console.log(items.length);      // 3

// template literals (string interpolation!):
console.log(`hello, ${player.name}! you have ${player.health}hp`);
JS · Chapter 03

Functions

✦

Functions are reusable blocks of code. Define once, call whenever.

functions
// function declaration
function greet(name) {
  return `hello, ${name}! ✦`;
}
console.log(greet("clio"));  // "hello, clio! ✦"

// arrow function (modern, concise)
const double = (n) => n * 2;
console.log(double(5));  // 10

// with multiple lines
const takeDamage = (player, amount) => {
  player.health -= amount;
  if (player.health <= 0) {
    return "game over!";
  }
  return `${player.name} has ${player.health}hp left`;
};

// default parameters
function cast(spell, power = 10) {
  return `casting ${spell} at power ${power}`;
}
cast("fireball");        // power defaults to 10
cast("fireball", 99);   // power is 99
JS · Chapter 04

The DOM

✦

The DOM (Document Object Model) is how JavaScript sees your HTML — as a tree of objects you can read, change, create, and delete.

dom manipulation
// SELECTING elements
const title  = document.querySelector("h1");         // first h1
const cards  = document.querySelectorAll(".card");     // all .card
const btn    = document.getElementById("submit-btn"); // by id

// READING & CHANGING content
title.textContent = "new title";          // change text
title.innerHTML   = "<em>fancy</em> title"; // change HTML inside
console.log(btn.textContent);             // read it

// CHANGING styles
title.style.color = "#c4607a";
title.style.fontSize = "3rem";

// CHANGING classes
title.classList.add("visible");
title.classList.remove("hidden");
title.classList.toggle("active");       // adds if absent, removes if present

// CHANGING attributes
const img = document.querySelector("img");
img.setAttribute("src", "new-image.png");
img.getAttribute("alt");

// CREATING & APPENDING new elements
const newDiv = document.createElement("div");
newDiv.textContent = "i appeared from thin air";
newDiv.classList.add("card");
document.body.appendChild(newDiv);

// REMOVING an element
newDiv.remove();
JS · Chapter 05

Events

✦

Events are things that happen — clicks, key presses, hovering, scrolling, the page loading. You "listen" for them with addEventListener.

event listeners
const btn = document.querySelector("button");

// click
btn.addEventListener("click", () => {
  console.log("clicked!");
});

// the event object gives you info about what happened
btn.addEventListener("click", (event) => {
  console.log(event.target);      // the element that was clicked
  event.preventDefault();        // stop default behavior (e.g. form submit)
});

// input: fires every time the value changes
const input = document.querySelector("input");
input.addEventListener("input", (e) => {
  console.log(e.target.value);    // current value of the input
});

// keyboard
document.addEventListener("keydown", (e) => {
  console.log(e.key);             // "Enter", "ArrowLeft", "a", etc.
  if (e.key === "Escape") closeModal();
});

// page ready
document.addEventListener("DOMContentLoaded", () => {
  console.log("page is ready!");
});

// scroll
window.addEventListener("scroll", () => {
  console.log(window.scrollY);    // pixels scrolled from top
});

Conditionals & Loops

✦
conditionals & loops
// IF / ELSE IF / ELSE
if (health <= 0) {
  console.log("game over");
} else if (health < 20) {
  console.log("critical — run!");
} else {
  console.log("you're okay");
}

// comparison operators
// === (equal)   !== (not equal)   >   <   >=   <=
// && (AND)   || (OR)   ! (NOT)

// ternary: shorthand if/else
const msg = health > 0 ? "alive" : "defeated";

// FOR loop
for (let i = 0; i < 5; i++) {
  console.log(`round ${i}`);
}

// FOR...OF: loop over arrays (so clean)
const items = ["sword", "potion", "map"];
for (const item of items) {
  console.log(item);
}

// WHILE loop
while (health > 0) {
  health -= 10;    // be careful of infinite loops!
}

// array methods (super useful)
items.forEach(item => console.log(item));
const doubled = [1,2,3].map(n => n * 2);    // [2,4,6]
const evens   = [1,2,3,4].filter(n => n % 2 === 0); // [2,4]
